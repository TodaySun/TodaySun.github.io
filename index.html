blog制作总结

发表于 2018-06-19

本次实训要求用hexo、github做个性化blog，经过一周的制作，blog已经完成，最后做一下总结。

总结

本次实训是基于大家都没有这方面的基础展开的，所以在制作过程中难免会遇到很多的问题，可能是细节上的，比如在改文件、改配置的时候，没有将该改的地方改完、或者是修改的格式不对，也有可能是在制作过程中某一个步骤遗漏掉了，没有进行操作。

如果这些问题得不到解决，就会导致进度变慢，完成blog的时间就会变慢，所以在制作过程中需要以再小心，多注意一下细节问题。

既然出现了问题，就要把他解决掉，或者是避免问题出现

制作博客中遇到的问题

发表于 2018-06-13

安装

首先是安装中遇到的问题…好像没遇到…跳过

终端

根据提供的参考博客，完成安装后在文件夹下打开gitbash运行以下代码:

hexo i blog //这个应该是创建根目录
cd blog //进入创建的根目录
hexo g //生成代码
hexo s //部署到本地

打开浏览器就可以查看：

可以在里面选择自己喜欢的主题

之后清楚一下缓存再生成代码部署到本地就ok了

hexo c
hexo g
hexo s

在打开浏览器就可以看到效果

上传到github

先注册一个github账号

https://github.com/ 这是github网址

进去后创建新项目，注意：Owner和Repository name要一致，只是Repository name要多一个后缀名 .github.io不然会导致下面不好继续操作

至于主题，可以用自己的，也可以在github上选一个自己喜欢的样式，自己的样式就需要上传到github了（在设置里面可以看到你自己的连接）

将你的链接配置到hexo站点文件中，按照

然后你就可以在浏览器上看到你自己的blog了

上传自己的github样式

在gitbash中配置账户信息(根目录下打开gitbash)：

git config --global user.name "你的用户名"
git config --global user.email "你的邮箱"

然后创建SSH

在gitbash中输入:

ssh-keygen -t rsa -C "youremail@example.com"

SSH生成后进入SSH:

cd ~/.ssh

进去后输入:

cat id_rsa.pub

后面的那一串就是密匙，回到github中配置密匙，就可以上传你自己的样式了

装上插件:在gitbash中

npm install hexo-deployer-git --save
hexo c
hexo g
hexo d

发布文章

在gitbash中

hexo n "文档名"

然后在根目录下找到source文件夹下面的_posts文件夹下面就是你创建的文档，参考自带的hellow-world就可以自己编写博客并发布了，也可以查阅markdown语言编写博客。

关于制作blog的一点体会

发表于 2018-06-12

个人体会

6月13日

本次实训，制作一个个人blog，从安装hexo到发布文章，个人感觉还是挺有成就感的，毕竟是自己亲手做出来的blog，中间也遇到很多的问题，不过通过老师的帮助，和在网上查阅资料解决了，做完blog后发现其实制作个人blog并不困难，只需要注意一些细节方面就可以做的很轻松，还要学会寻找错误，因为大部分的错误都会在gitbash里面显示出来，所以只要好好的看一下，很多的地方都能解决掉.

在制作blog过程中，最容易出现的问题就是，在gitbash中创建项目的时候没有注意格式，前面的名称和后面的名称不一致，会导致后面操作困难，又找不到错误的地方

在生成密匙后要记得安装插件，注意修改你的站点配置文件，在修改的时候注意格式

后面的发布文章，编写的时候用的是markdown语言，可以在网上参考相关的资料了解一下
